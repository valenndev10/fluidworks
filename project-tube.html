02

Región de entrada hidrodinámica en tubo finito

Problema clásico de validación: simulación del desarrollo del perfil de velocidad en un tubo cilíndrico, desde el perfil uniforme de entrada hasta el perfil parabólico de Poiseuille completamente desarrollado. Validación contra la solución analítica conocida.

COMSOL Multiphysics 2D Axisimétrico Flujo laminar (spf) ρ = 1, μ = 1
Fase A · Configuración · Fase B · Validación · Errores
▸ Fase A

Configuración del modelo

Crear el componente 2D axisimétrico

Nuevo modelo → dimensión: 2D axisimétrico (no 2D plano). Esto es fundamental: en 2D-axi COMSOL entiende un tubo cilíndrico con simetría de revolución. Las coordenadas del marco espacial deben ser r, phi, z (sistema cilíndrico). Si fueran x, y, estaría resolviendo un canal entre placas paralelas, que es un problema diferente.

Física: Flujo laminar (spf). Estudio: Estacionario.

Verificación: Para confirmar que el componente es 2D-axi y no plano, hacer clic en el nodo del Componente. Las coordenadas del marco espacial deben decir r, phi, z (sistema cilíndrico). Si dice x, y, es 2D plano — crear un componente nuevo.

Parámetros globales

Definiciones globales → Parámetros:

radio = 0.1[m] // Radio del tubo D = 2*radio // Diámetro rho = 1[kg/m^3] // Densidad (simplificada) mu = 1[Pa*s] // Viscosidad dinámica (simplificada) Re = 100 // Número de Reynolds u_e = Re*mu/(rho*D) // Velocidad de entrada (derivada de Re) L = 1.5*0.05*Re*D // Longitud del tubo (1.5 × Le teórica)
¿Por qué ρ = 1, μ = 1? Con estas propiedades simplificadas, el Reynolds queda Re = ue·D, lo que permite controlarlo fácilmente cambiando solo el parámetro Re. No simulamos un fluido real — solo importa que el número adimensional sea correcto. El profesor lo recomendó como práctica habitual en problemas académicos de validación.
Longitud del tubo: La longitud de entrada teórica es Le ≈ 0.05·Re·D. Para Re = 100, esto da Le = 1 m. Usando L = 1.5·Le nos aseguramos de que el flujo tenga espacio de sobra para desarrollarse completamente. Si Re es demasiado bajo (por ejemplo Re = 0.2 con los parámetros iniciales), la longitud de entrada es de milímetros y el fenómeno apenas se aprecia.

Geometría

Geometría → Rectángulo:

  • Anchura: radio
  • Altura: L
  • Base: Esquina en (0, 0), rotación: 0°

El borde izquierdo (r = 0) es el eje de simetría axial. El borde derecho (r = radio) es la pared del tubo. No es necesario modelar una región más ancha antes del tubo: la condición de velocidad uniforme en la entrada ya representa implícitamente que el fluido viene de un depósito grande.

Propiedades del fluido

En lugar de añadir un material de la biblioteca, se definen directamente en Propiedades de fluidos 1: ρ = rho, μ = mu.

Condiciones de contorno

  • Entrada (borde inferior, z = 0) → Velocidad normal de entrada: u_e
  • Salida (borde superior, z = L) → Presión: 0 Pa, con "Suprimir reflujo" activado
  • Pared (borde derecho, r = radio) → No deslizamiento (por defecto)
  • Eje de simetría (r = 0) → COMSOL lo aplica automáticamente en modelos 2D-axi
Sobre la entrada: Usar "Velocidad normal de entrada" y no "Campo de velocidad". Con "Campo de velocidad" hay que especificar las componentes r, phi, z manualmente y es fácil confundirse (poner la velocidad en la componente r en vez de z haría que el fluido entre radialmente).

Mallar y resolver

Mallado fino estándar (triángulos libres). Resolver en estacionario. Con la geometría y parámetros bien definidos, la solución converge en segundos.

▸ Fase B

Validación de resultados

Comprobación del modelo contra las soluciones analíticas conocidas para el perfil de Poiseuille y la caída de presión.

Gráfico 2D de velocidad

Resultados → Gráfico de superficie con la componente w (velocidad axial). Se observa claramente cómo el fluido entra con velocidad uniforme (colores azulados/verdes) y progresivamente se acelera en el centro (rojo) mientras se frena cerca de la pared (azul). Es el desarrollo del perfil de velocidad en acción.

Perfil de velocidad en la salida

Resultados → Grupo gráfico 1D → Gráfico lineal:

  • Expresión: w (componente axial)
  • Contorno: borde de la salida (z = L)
Resultado: perfil parabólico con wmax = 2 · ue en r = 0 y w = 0 en r = radio ✓

Perfil de Poiseuille validado contra la solución analítica.

Atención a las variables: En 2D axisimétrico, las componentes de velocidad son u (radial), v (azimutal), w (axial). La velocidad axial del flujo es w, no v. Si los resultados salen con la variable incorrecta, verificar el orden de coordenadas (r, phi, z). Si los gráficos heredados de un modelo anterior no funcionan, recrearlos desde cero.

Caída de presión a lo largo del eje

Grupo gráfico 1D → Gráfico lineal:

  • Expresión: p (presión)
  • Contorno: eje de simetría (r = 0)

Se observan dos zonas claramente diferenciadas:

  • Región de entrada: caída más pronunciada y no lineal (el fluido se está acelerando en el centro)
  • Flujo desarrollado: caída lineal (gradiente de presión constante de Poiseuille)

Verificación analítica del gradiente en la zona desarrollada:

dp/dz = −8μue / R² = −8 · 1 · ue / (0.1)² = −800 · ue Pa/m

Consistente con los valores obtenidos en la simulación. ✓

Resumen de la validación

El modelo reproduce correctamente tanto el perfil de velocidad de Poiseuille como la caída de presión teórica, validando la configuración de COMSOL para problemas de flujo interno con desarrollo de capa límite.

▸ Errores y soluciones

Problemas encontrados durante el proyecto

Error

Componente 2D plano en vez de 2D axisimétrico

Si el modelo se crea como 2D plano, COMSOL resuelve un canal entre placas paralelas infinitas en lugar de un tubo cilíndrico. Los resultados parecen razonables pero son físicamente incorrectos para el problema del tubo.

Solución

Verificar que las coordenadas del marco espacial sean r, phi, z (no x, y). Si están mal, crear un componente nuevo como 2D axisimétrico desde el inicio y replicar la configuración.

Error

Velocidad de entrada en la componente radial en vez de axial

Al usar "Campo de velocidad" en la condición de Entrada, se puso la velocidad en la componente r (radial). Esto hace que el fluido entre radialmente en vez de en la dirección del tubo.

Solución

Cambiar a "Velocidad normal de entrada". Con esta opción solo se indica la magnitud y COMSOL orienta la velocidad perpendicular a la frontera automáticamente. Mucho más limpio y menos propenso a errores.

Error

Coordenadas r, phi, z en orden incorrecto

Las coordenadas estaban como r, z, phi en vez de r, phi, z. Esto intercambia el significado de las componentes de velocidad: w (que debería ser axial) pasa a ser azimutal, y el perfil parabólico aparece en la variable incorrecta.

Solución

Corregir el orden de coordenadas a r, phi, z en la configuración del componente. Después de corregir, recalcular la solución y recrear los gráficos de resultados.

Error

Gráficos heredados de un modelo anterior no muestran datos correctos

Al cambiar de componente (de 2D plano a 2D-axi), los grupos gráficos creados anteriormente no se actualizan correctamente y muestran resultados planos o erróneos.

Solución

Eliminar los gráficos heredados y crearlos de nuevo desde cero. Cuando se cambia de componente, siempre recrear también los gráficos de resultados.

Error

La región de entrada es imperceptible (Reynolds demasiado bajo)

Con los parámetros iniciales (ue = 1, ρ = 1, μ = 1, D = 0.2), el Reynolds era solo 0.2. La longitud de entrada teórica era Le = 0.002 m (2 mm), haciendo el fenómeno invisible en un tubo de 5 metros.

Solución

Parametrizar el modelo usando el Reynolds como variable de control: u_e = Re*mu/(rho*D). Con Re = 100, Le = 1 m y el fenómeno se aprecia claramente. Ajustar L a ~1.5·Le para un modelo eficiente.
