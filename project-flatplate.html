03

Capa límite térmica sobre placa plana

Simulación 2D de la capa límite hidrodinámica y térmica sobre una placa plana con flujo paralelo uniforme. Validación contra Blasius y Pohlhausen, seguida de estudios paramétricos variando Re y Pr.

COMSOL Multiphysics 2D Estacionario Flujo laminar + Heat Transfer Non-Isothermal Flow
Fase A · Configuración · Fase B · Blasius (Pr = 1) · Fase C · Pohlhausen y cambio a aire · Fase D · Paramétrico · Errores
▸ Fase A

Configuración del modelo (propiedades simplificadas)

Configuración inicial con propiedades artificiales (ρ = 1, μ = 0.01) para tener control directo del Reynolds. La idea era empezar simple, validar Blasius, y después pasar a la parte térmica.

Crear el modelo y seleccionar las físicas

Nuevo modelo → 2D. Añadir las físicas:

  • Flujo laminar (spf) — resuelve Navier-Stokes
  • Transferencia de calor en fluidos (ht) — resuelve la ecuación de energía

Después de añadir ambas, ir a Multifísica y añadir manualmente Flujo no isotérmico (Non-Isothermal Flow). Este nodo acopla ambas físicas.

Estudio: Estacionario.

Parámetros globales (versión inicial)

U_inf = 1 // Velocidad (se irá subiendo) T_inf = 293.15[K] // Temperatura del flujo libre (20°C) T_p = 333.15[K] // Temperatura de la placa (60°C) L_placa = 1[m] // Longitud de la placa L_pre = 0.5[m] // Longitud zona pre-placa H = 2[m] // Altura del dominio

Geometría: dos rectángulos adyacentes

  • Rectángulo 1 (zona pre-placa): anchura = L_pre, altura = H, esquina en (0, 0)
  • Rectángulo 2 (zona de la placa): anchura = L_placa, altura = H, esquina en (L_pre, 0)

La zona pre-placa permite que el flujo llegue con perfil uniforme al borde de ataque de la placa.

Propiedades del fluido (simplificadas)

Definidas directamente en Propiedades de fluidos y Fluido 1 de Heat Transfer:

ρ = 1 [kg/m³] // Densidad μ = 0.01 [Pa·s] // Viscosidad → ν = 0.01 Cp = 1 [J/(kg·K)] // Capacidad calorífica k = 0.01 [W/(m·K)] // Conductividad térmica

Con esto: ν = 0.01, α = 0.01, Pr = 1. Reynolds: ReL = U_inf / 0.01 = 100·U_inf.

Condiciones de contorno

Flujo laminar: Entrada (U_inf, borde izq.), Salida (p = 0, borde der.), Simetría (borde sup.), Pared no-slip (borde inf. rect. 2), Flujo deslizante (borde inf. rect. 1).

Heat Transfer: T_inf en entrada, T_p en la placa, outflow en salida.

Mallado con capas de contorno

Tamaño global: "Fino". Triangular libre + Capas de contorno sobre la placa: primera capa = 0.005 m, ~15 capas, factor 1.2.

Resolver: escalonamiento de U_inf

El solver no converge directamente a velocidades altas. Se escalona manualmente:

  • U_inf = 1 (Re = 100) — converge fácilmente
  • Subir a U_inf = 10 (Re = 1000) — usando la solución anterior como arranque
  • Intentar U_inf = 100 (Re = 10000) — converge pero la malla no resuelve bien la capa límite
▸ Fase B

Validación hidrodinámica — Blasius (propiedades simplificadas)

Comparación del perfil de velocidad numérico con la solución analítica de Blasius usando ν = 0.01, Pr = 1.

Línea de corte y perfil adimensional

Línea de corte vertical a x = 0.75, de y = 0 a y = 0.1. Gráfico lineal:

  • Eje Y: u/u_inf
  • Eje X: y*sqrt(u_inf/(0.01*(x-L_pre)))
¿Por qué adimensionalizar? El gráfico dimensional (u vs y) dice "tiene buena pinta". El adimensional (u/U∞ vs η) dice "mi simulación se desvía un 6%". Solo lo segundo es una validación.

Comparación en Python con scipy

Exportar datos a .txt. En Python, resolver Blasius (ODE 3er orden) con scipy.integrate.solve_ivp, shooting con f''(0) ≈ 0.3321. Superponer curvas.

Re = 80 → error 24% | Re = 1000 → error 6%

Con las propiedades iniciales (ν = 1, U_inf = 1), Re = 80. El error era ~24% con overshoot del 12%: la capa límite ocupaba δ/L ≈ 56%, demasiado gruesa para Blasius.

Tras cambiar μ a 0.01 (ν = 0.01) y subir U_inf a 10 (Re = 1000), el acuerdo mejoró a ~6%. Se intentó Re = 10000 (U_inf = 100) pero la malla no tenía resolución suficiente.

▸ Fase C

Validación térmica — el camino a Pohlhausen

La validación del perfil de temperatura resultó mucho más difícil de lo esperado. Con Cp = 1, la disipación viscosa corrompía los resultados. La solución fue cambiar a aire real.

Primer intento: θ vs η con Pr = 1

Con Pr = 1, los perfiles de velocidad y temperatura deberían ser idénticos. Se graficó θ = (T−T_p)/(T_inf−T_p) vs η.

Resultado: θ solo llegaba a ~0.78 en vez de 1.0. La temperatura lejos de la placa era ~302 K en vez de 293 K — un exceso de 9 K que no debería estar.

Diagnóstico: disipación viscosa (Brinkman = 25)

El nodo de Multifísica incluye por defecto el calentamiento por fricción μ·(du/dy)².

Br = μ·U² / (k·ΔT) = 0.01 · 100 / (0.01 · 40) = 25 ≫ 1

Con Cp = 1, la energía cinética U²/(2·Cp) = 50 K — comparable al ΔT de 40 K del problema.

Intentos de corrección (ninguno funcionó)

  • Desactivar disipación viscosa en Multifísica → no mejora suficiente
  • Añadir condición T = T_inf en el borde superior → no mejora
  • Subir Cp a 100 y k a 1 (manteniendo Pr = 1) → tampoco

Con propiedades artificiales tan alejadas de la realidad, cualquier término residual de energía cinética genera artefactos térmicos.

Solución definitiva: cambiar a aire real

  • Materiales → Añadir material → "Air" → asignar a ambos dominios
  • Flujo laminar y Heat Transfer → cambiar propiedades a "Desde material"
  • Ajustar U_inf para Re = 1000: νaire ≈ 1.516×10⁻⁵ → U_inf = 0.015 m/s
  • Actualizar η: y*sqrt(u_inf/((spf.mu/spf.rho)*(x-L_pre)))

Con aire, Cp ≈ 1005 y la energía cinética es despreciable. Pr = 0.71, lo que hace la comparación más interesante.

Validación con aire: Blasius + Pohlhausen

Se exportaron ambos perfiles y se compararon en Python. Con Pr = 0.71, el perfil de velocidad (azul) sube más rápido que el de temperatura (verde): a un η dado, la velocidad ha recuperado más porcentaje del valor libre que la temperatura.

Interpretación: Pr = 0.71 < 1 → α > ν → el calor se difunde más lejos → δT > δ → θ(η) crece más lento que f'(η).

▸ Fase D

Estudios paramétricos — Re y Pr

Dos barridos independientes con aire real para estudiar el efecto del Reynolds y del Prandtl sobre los perfiles y el número de Nusselt.

Operador de integración para el Nusselt

Definiciones → Operadores de componente → Integración (tipo Contorno) sobre la placa. Nombre: intop1.

Integra una expresión a lo largo del contorno. Por ejemplo, intop1(ht.ntflux) da el flujo de calor total. El Nusselt promedio:

NuL = intop1(ht.ntflux) / (T_p − T_inf) / k_aire
Sin L_placa (la integración ya cubre la longitud). Valores negativos → tomar absoluto. Se usó un parámetro k_aire = 0.0257 porque ht.k no estaba definido.

Estudio 1: Variar Re (barrido de U_inf)

U_inf = 0.0015 0.005 0.015 0.05 0.15 [m/s] Re_L ≈ 100 330 1000 3300 10000

Estudio 2: Variar Pr (barrido de k)

Fijar U_inf = 0.015 (Re = 1000). Parámetro Pr_param. En Heat Transfer → Fluido 1, k como "Definido por el usuario":

k = spf.mu * 1005 / Pr_param Pr = 0.1 0.5 0.71 2.0 7.0 (metales líquidos → aire → agua)

▸ Errores y soluciones

Problemas encontrados durante el proyecto

Error

Física configurada como Creeping Flow en vez de Laminar Flow

Se seleccionó "Flujo reptante" (Stokes) que omite términos inerciales. Sin inercia no hay capa límite.

Solución

Eliminar Creeping Flow, añadir Flujo laminar (spf). Verificar que "Omitir inercia" esté desactivado.

Error

"Gas ideal" en el nodo de Transferencia de calor

Fluido 1 tenía "Gas ideal", introduciendo dependencias de temperatura y fallos de convergencia.

Solución

Cambiar a "Gas" o "Líquido" (propiedades constantes).

Error

Solver segregado no converge al subir U_inf

Al cambiar U_inf directamente de 1 a 10 o 100, el campo inicial está demasiado lejos de la solución.

Solución

Escalonar: resolver con U_inf = 1, subir a 5, 10... Cada solución sirve como arranque para la siguiente.

Error

Expresión de η no actualizada al cambiar μ

Al pasar de μ = 1 a μ = 0.01, la expresión seguía usando ν = 1. Valores de η 10× menores.

Solución

Usar y*sqrt(u_inf/((spf.mu/spf.rho)*(x-L_pre))) para que se actualice automáticamente.

Error

Validación a Re = 80: error del 24%

δ/L ≈ 56% — la hipótesis de capa límite delgada no se cumple.

Solución

Subir Re a 1000 (μ = 0.01, U_inf = 10). Error baja a ~6%.

Error

Perfil térmico no alcanza θ = 1 (Brinkman = 25)

Con Cp = 1, la disipación viscosa genera calentamiento espurio de ~9 K. Se intentó desactivar disipación, subir Cp, forzar T_inf arriba — nada funcionó.

Solución

Cambiar a aire real. Con Cp ≈ 1005, la energía cinética es despreciable. U_inf = 0.015 m/s para mantener Re = 1000.

Error

Malla insuficiente a Re = 10000

Capa límite de ~0.035 m, primera celda de 0.005 m demasiado gruesa.

Solución

Quedarse con Re = 1000 como punto dulce. Mostrar en el paramétrico cómo varía el acuerdo — el "punto dulce" es un cruce de error teórico y numérico específico de la malla.

Error

Tabla de Nu del barrido en Pr muestra valores del barrido de Re

La Evaluación global seguía apuntando al conjunto de datos del barrido de U_inf.

Solución

En Datos → "Selección de parámetros", seleccionar Pr_param en vez de u_inf. Reevaluar.

Error

Nu calculado con k_aire fijo en vez de k variable con Pr

La expresión usaba k_aire = 0.0257 constante, pero k cambia con cada Pr. Los Nu bajaban al subir Pr.

Solución

Corregir en Python: Nucorrecto = NuCOMSOL × k_aire / kreal(Pr).
